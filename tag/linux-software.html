Software

  • Pycharm

    • to view the .idea directory and/or the workspace.xml file:

    Pycharm won't show this directory when default "Project" scope is used. You can switch this to "Project Files" and .idea directory will be revealed.

  • Miniconda

    • install Miniconda

      $ wget https://repo.continuum.io/miniconda/Miniconda3-latest-Linux-x86_64.sh
$ bash Miniconda3-latest-Linux-x86_64.sh

    • check which venvs are currently installed: conda info --envs

    • conda info will tell you what environment is currently active (under “default environment”)

      read more

  • Jupyter

    • make sure to create a virtual environment with the necessary modules (i.e. the notebook's imports)
    • activate the venv and run pip install jupyterlab (or add it to the venv setup script)
    • do not run the notebook from pycharm! use the command line: $ jupyter notebook

  • TeXworks (MikTex)

    • If you are having issues with missing packages when compiling pdfLaTeX+Makeindex+BibTeX run

      $ apt-get install texlive-latex-extra

    related help

  • GraphViz

    • To view a .gv graphviz file run below command to create a viewable pdf:
      $ f = Source.from_file('file-name.gv')
$ f.render()

  • RoboMongo

    $ cd /opt
$ sudo wget https://download.robomongo.org/1.0.0/linux/robomongo-1.0.0-linux-x86_64-89f24ea.tar.gz
$ sudo tar -zxvf robomongo-1.0.0-linux-x86_64-89f24ea.tar.gz
$ sudo rm robomongo-1.0.0-linux-x86_64-89f24ea.tar.gz
