Azure Solutions and Management Tools


Core Azure Solutions

IoT SaaS Solutions

  • IoT Hub and IoT Central

    • high speed, low latency;
    • e.g. 1M thermostats around the world
    • millions of small pieces of data coming from these devices

  • Azure Sphere

    • both a chip AND an OS
    • security protocol for IoT devices

Big Data and Analytics Solutions

  • Azure Synapse Analytics

    • Formerly SQL Data Warehouse

  • HD Insight

    • Umbrella for all Apache big data solutions
    • Microsoft-managed Hadoop services
    • Micorsoft operates these services and keep them running and you interact with them in a SaaS or PaaS way
    • Hadoop Apache Spark, Hive, Kafka, Storm

  • Azure Databricks

    • Open-source data analytics platform
    • data analysts, db admins, business stakeholders, technical side, reporting side, working in one workspace to make sense of the data
    • JuPyter notebook > run big data queries and then annotate the results so that it becomes a report i.e. real-time reporting that comes with reports, data and handwritten notes of the data analysts

AI Solutions

  • These are Microsoft-developed services
  • Azure Machine Learning,
    • APIs for
      • images, vision and; face, place and object recognition
      • audio-to-text, text-to-speech
      • translation
      • predictions, categorisation, likelihood of customers …
  • Cognitive Services: ??
  • Azure Bot Service

Serverless computing

  • Azure Functions (see video 31 @ 1:40 for DEMO)

    • Small pieces of code hosted in Azure that run after some trigger e.g. timer, data change, file being stored
    • Called serverless because no instance involved
      • you don’t define nbr of CPUs, size of RAM, storage
      • you don’t chose a hosting plan like you do in a web app
    • Consumption-based billing
    • see “Functions Editor”
    • Search “Function” in marketplace

  • Logic Apps

    • Is a workflow service
    • A logic app can call an Azure Function we created!
    • e.g. when a new file is created in dropbox, copy it to one drive or get daily reminders emailed to you or after executing function, store file on Azure Blob file storage …
    • Has lots of “external connectors”
    • Consumption-based billing
    • Search “logic” in marketplace
    • see video 32 @ 0:50 for DEMO

  • Event Grid

    • Messaging communication with the outside world
    • Configurable but not programmable

DevOps Solutions

  • Azure DevOps

    • Tie in development functions (scripts, automation) into your company’s operations (IT infrastructure) e.g. CI/CD
    • See DevOps pipelines, Agile development processes

  • Github

    • See Github Actions:
    • allows to push code from Github into Azure automatically when source code changes (CI)
    • check for errors (similar to pre-commit)

  • Azure DevTest Labs: ??


Azure Management Tools

  • Azure Portal, Azure PowerShell, Azure CLI

  • Cloud Shell

    • if you don’t want to install PowerShell etc
    • See video 34 @ 2:00 for DEMO

  • Azure Mobile App to check resource status, stop/start resources etc

  • Azure Advisor

    • 5 categories:
      • High availability
      • Security
      • Performance
      • Operational excellence
      • Cost
    • see video 35 @ 1:20 for DEMO

  • Azure Resource Manager (ARM) templates

    • IS A JSON file
    • All actions that you take to manage your Azure resources goes through the ARM layer
    • Infrastructure as Code
      • store all of the infrastructure information in a JSON file as an ARM template and store it within github (see video 36 @2:50)
    • Every time someone wants to change the infrastructure they have to change the template, then re-deploy
    • It’s a way of documenting your infrastructure and how it changes

  • Azure Monitor

    • to monitor your own resources
    • Useful if you have hundreds of resources
    • Centralised dashboard for (most of) your resources / services
    • Can create customised dashboards

  • Azure Service Health

    • Azure Monitor for Azure as a whole i.e. for regions or services etc
    • Issues that affect all regions or all services / resources (that you also happen to be using)
    • Can add it to your dashboard
    • Has past issues
    • Ability to do alerts (SMS, Email, …)
